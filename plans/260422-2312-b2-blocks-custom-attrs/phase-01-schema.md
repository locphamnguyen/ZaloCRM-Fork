# Phase 01 — Schema & Migrations

**Status:** pending | **Priority:** P1 | **Effort:** 4h | **Blocks:** none

## Context
Foundation for B2. Adds 5 tables + 1 column. No changes to existing rows except `Contact.custom_attrs` default `'{}'`.

## Requirements
- All tables org-scoped (`org_id` FK to `organizations`, `onDelete: Cascade`)
- Reversible migration
- Indexes for common query patterns
- JSONB columns where flexibility needed; structured columns where queryable

## Files to Modify / Create
- MODIFY: `backend/prisma/schema.prisma` — append models below; add column to `Contact`
- CREATE: `backend/prisma/migrations/<timestamp>_b2_blocks_attrs/migration.sql` — generated by `prisma migrate dev --name b2_blocks_attrs`

### Block
```prisma
model Block {
  id          String   @id @default(uuid())
  orgId       String   @map("org_id")
  ownerUserId String?  @map("owner_user_id")
  name        String
  type        String   // text | html | image | video | file | link | card
  content     Json     // type-specific payload (see below)
  variables   Json     @default("[]")  // detected vars cache for UI hints
  isShared    Boolean  @default(false) @map("is_shared")
  deletedAt   DateTime? @map("deleted_at")
  createdAt   DateTime @default(now()) @map("created_at")
  updatedAt   DateTime @updatedAt @map("updated_at")

  org         Organization @relation(fields: [orgId], references: [id], onDelete: Cascade)
  owner       User?        @relation(fields: [ownerUserId], references: [id])
  attachments BlockAttachment[]

  @@index([orgId, type, deletedAt])
  @@index([orgId, ownerUserId])
  @@map("blocks")
}
```

### BlockAttachment
```prisma
model BlockAttachment {
  id        String   @id @default(uuid())
  blockId   String   @map("block_id")
  orgId     String   @map("org_id")  // denormalized for org isolation queries
  kind      String   // image | video | file
  filename  String
  mimeType  String   @map("mime_type")
  sizeBytes Int      @map("size_bytes")
  storagePath String @map("storage_path")  // /<orgId>/blocks/<blockId>/<uuid>-<filename>
  position  Int      @default(0)
  createdAt DateTime @default(now()) @map("created_at")

  block Block @relation(fields: [blockId], references: [id], onDelete: Cascade)

  @@index([blockId, position])
  @@map("block_attachments")
}
```

### CustomAttributeDefinition
Per-org schema for what custom attrs exist + their types/validation.
```prisma
model CustomAttributeDefinition {
  id          String   @id @default(uuid())
  orgId       String   @map("org_id")
  key         String   // e.g. "nhu_cau" — snake_case, [a-z0-9_]
  label       String   // human display
  dataType    String   @map("data_type") // string | number | date | boolean | enum
  enumValues  Json?    @map("enum_values") // ["mua","thue"] when dataType=enum
  required    Boolean  @default(false)
  deletedAt   DateTime? @map("deleted_at")
  createdAt   DateTime @default(now()) @map("created_at")
  updatedAt   DateTime @updatedAt @map("updated_at")

  org Organization @relation(fields: [orgId], references: [id], onDelete: Cascade)

  @@unique([orgId, key])
  @@map("custom_attribute_definitions")
}
```

### ApiKey
```prisma
model ApiKey {
  id           String    @id @default(uuid())
  orgId        String    @map("org_id")
  name         String    // user-facing label
  keyPrefix    String    @map("key_prefix")  // first 8 chars for display
  keyHash      String    @unique @map("key_hash") // sha256(secret)
  scopes       Json      @default("[\"read\"]")  // ["read","write"]
  lastUsedAt   DateTime? @map("last_used_at")
  expiresAt    DateTime? @map("expires_at")
  revokedAt    DateTime? @map("revoked_at")
  createdByUserId String? @map("created_by_user_id")
  createdAt    DateTime  @default(now()) @map("created_at")

  org     Organization @relation(fields: [orgId], references: [id], onDelete: Cascade)
  creator User?        @relation(fields: [createdByUserId], references: [id])

  @@index([orgId, revokedAt])
  @@map("api_keys")
}
```

### ApiKeyUsage (rate-limit tracking, optional but cheap)
```prisma
model ApiKeyUsage {
  id        String   @id @default(uuid())
  apiKeyId  String   @map("api_key_id")
  bucket    DateTime // truncated to minute
  count     Int      @default(1)

  @@unique([apiKeyId, bucket])
  @@map("api_key_usage")
}
```

## Block.content JSON shapes (documented, validated in P02)
- text: `{ "text": "..." }`
- html: `{ "html": "..." }`
- image: `{ "caption": "...", "attachmentIds": [...] }`
- video: `{ "caption": "...", "attachmentIds": [...] }`
- file: `{ "caption": "...", "attachmentIds": [...] }`
- link: `{ "url": "...", "title": "...", "thumbnail": "..." }`
- card: `{ "imageAttachmentId": "...", "title": "...", "description": "...", "ctaText": "...", "ctaUrl": "..." }`

## Implementation Steps
1. Add models + Contact column to `schema.prisma`
2. Run `npx prisma migrate dev --name b2_blocks_attrs` from `backend/`
3. Add raw SQL to migration for GIN index on `contacts.custom_attrs` (Prisma may not emit it; verify):
   ```sql
   CREATE INDEX IF NOT EXISTS idx_contacts_custom_attrs_gin ON contacts USING gin (custom_attrs jsonb_path_ops);
   ```
4. Run `npx prisma generate`
5. Verify backward compat: `npx prisma migrate status` clean; existing seed scripts run
6. Manual rollback test: `prisma migrate reset` on local DB; re-apply

## Success Criteria
- [ ] `npx prisma migrate deploy` green on fresh DB
- [ ] Existing data unchanged (verified by row counts pre/post on staging clone)
- [ ] All FK cascades work (delete org → cascades to blocks/attrs/keys)
- [ ] GIN index visible: `\d contacts` shows it
- [ ] Generated client compiles: `tsc --noEmit` in backend

## Risks
- Prisma JSONB GIN index syntax may need raw SQL — verified manually
- Migration on prod with large contacts table: GIN index build may take minutes — schedule during low-traffic window; monitor via `pg_stat_progress_create_index`

## Rollback
`prisma migrate resolve --rolled-back <migration_name>` + manual `DROP TABLE` for new tables and `ALTER TABLE contacts DROP COLUMN custom_attrs`. Test on staging before prod.
